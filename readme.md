	файл cart_actions.php:
		це скрипт для обробки дій з кошиком, що зберігається в сесії ($_SESSION['cart']).
		Ініціалізує $_SESSION['cart'], якщо кошик ще не існує.
		Підтримує зміну кількості товару через GET-параметри: qty (ключ елемента) та action (inc/dec).
		При зміні кількості відповідає JSON для AJAX-запитів (перевіряє HTTP_X_REQUESTED_WITH).
		Підтримує видалення одиночного елемента через GET-параметр remove.
		Підтримує очищення всього кошика через GET-параметр clear.

	файл index_actions.php:
		це API-ендпойнт, який обробляє POST-запити для додавання товарів до кошика (повідомлення у форматі JSON).
		Виконує session_start() і повертає Content-Type: application/json.
		Ініціалізує $_SESSION['cart'], якщо кошик не існує.
		Очікує POST-параметр action; підтримувані дії:
			1. add_pizza: очікує id, необов'язково name, price, img. Додає та інкрементує піцу у кошику під ключем pizza_{id}.
			2. add_drink: очікує drink_key, необов'язково name, price. Додає та інкрементує напій у кошику під ключем drink_{drink_key}.
		На кожну успішну дію повертає JSON з success: true і повідомленням; при помилці — success: falsе та message.
		Для невідомої дії повертає помилку Невідома дія.

    Примітка:
	    Обидва файли працюють на основі сесійного масиву $_SESSION['cart'] і не здійснюють валідації чи зберігання в базі даних.
	    index_actions.php розрахований на отримання даних через AJAX з фронтенду (повертає JSON).
